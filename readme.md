# [MVP Summit 2026](https://mvp.microsoft.com)

## 🔥 End-to-End Agentic DevOps with Azure MCP — Ship, Harden, Break, Investigate

**Hands-On Lab | Level: 300**

AI can deploy your app to Azure in 5 minutes. But should you trust what it built? In this lab, you'll use GitHub Copilot CLI with Azure skills to deploy a live Container App — then put on your architect hat and evaluate the AI's decisions. You'll review generated Bicep, identify what's missing for production, direct the AI to harden the deployment, break the app on purpose, and run a full forensic investigation — all without opening the Azure Portal.

---

### 🚀 Getting Started

1. **Clone the repo**
   ```bash
   git clone https://github.com/microsoft/mvp26-LAB011-end-to-end-agentic-devops-with-azure-mcp.git
   cd mvp26-LAB011-end-to-end-agentic-devops-with-azure-mcp
   ```

2. **Follow the lab instructions** — Start with the [Prerequisites](lab-instructions/01-prerequisites.md), then work through [Login & Launch](lab-instructions/02-login-and-launch.md) and the four scenarios:

   | Scenario | What You'll Do | Guide |
   |----------|---------------|-------|
   | 1. Ship & Harden | Deploy to Azure Container Apps, then audit for production gaps | [Scenario 1](lab-instructions/04-scenario-1-ship-and-harden.md) |
   | 2. See & Evaluate | Generate and critically review an architecture diagram | [Scenario 2](lab-instructions/05-scenario-2-see-and-evaluate.md) |
   | 3. Break & Triage | Introduce a failure and watch AI diagnose it | [Scenario 3](lab-instructions/06-scenario-3-break-and-triage.md) |
   | 4. Investigate & Operationalize | Run a post-mortem with KQL and set up alerts | [Scenario 4](lab-instructions/07-scenario-4-investigate-and-operationalize.md) |

   Full step-by-step instructions: [Lab Instructions](lab-instructions/00-overview.md)

### 🧠 Learning Outcomes

By the end of this lab, you will be able to:

- Explain how Azure **skills** chain together — one prompt can trigger `azure-prepare` → `azure-validate` → `azure-deploy` automatically
- Identify where AI-generated infrastructure gets you to 80% — and articulate the production gaps you need to close
- Critically review AI-generated Bicep, Dockerfiles, and architecture diagrams
- Describe how `azure-diagnostics` follows a diagnostic reasoning chain: hypothesis formation, log retrieval, correlation, root cause identification — mirroring how a senior SRE would triage
- Evaluate when to trust AI decisions and when to override them
- Write and modify KQL queries and alert rules generated from natural language prompts

### 💻 Technologies Used

1. [GitHub Copilot CLI](https://docs.github.com/en/copilot/github-copilot-in-the-cli) with [Azure MCP Server](https://learn.microsoft.com/azure/developer/azure-mcp-server) — AI-driven command-line interface with Azure skills for deployment, diagnostics, RBAC, and observability
1. [Azure Container Apps](https://learn.microsoft.com/azure/container-apps/overview) — Serverless container hosting platform for running the deployed application
1. [Azure Container Registry](https://learn.microsoft.com/azure/container-registry/container-registry-intro) — Private Docker image registry for storing and pulling container images
1. [Bicep](https://learn.microsoft.com/azure/azure-resource-manager/bicep/overview) — Azure's domain-specific Infrastructure as Code language, generated by AI and reviewed by you
1. [Azure Developer CLI (azd)](https://learn.microsoft.com/azure/developer/azure-developer-cli/overview) — Developer workflow orchestration for provisioning and deploying Azure resources
1. [Node.js](https://nodejs.org/) — Runtime for the starter application (lightweight HTTP API)
1. [Docker](https://www.docker.com/) — Container building and local testing
1. [Log Analytics](https://learn.microsoft.com/azure/azure-monitor/logs/log-analytics-overview) with [Kusto Query Language (KQL)](https://learn.microsoft.com/kusto/query/) — Log aggregation and querying for diagnostics and observability
1. [Azure RBAC](https://learn.microsoft.com/azure/role-based-access-control/overview) and [Managed Identity](https://learn.microsoft.com/entra/identity/managed-identities-azure-resources/overview) — Least-privilege access control and passwordless authentication for Azure resources

### 💬 Keep Learning with Copilot

Try these prompts with GitHub Copilot to explore the topics from this lab. Open Copilot Chat in Visual Studio Code or use Copilot CLI, and connect the [Microsoft Learn MCP Server](#-microsoft-learn-mcp-server) for the latest official documentation.

1. Understand Azure skills:

   ```
   Using the Microsoft Learn MCP Server, explain how Azure MCP Server skills like azure-prepare, azure-validate, and azure-deploy work together to deploy an application. What does each skill do?
   ```

2. Go deeper on infrastructure review:

   ```
   I have a Bicep template that deploys an Azure Container App. Review it for production readiness — check for managed identity, health probes, VNet integration, and diagnostic settings. What's missing?
   ```

3. Build your own observability:

   ```
   Help me write a KQL query that monitors an Azure Container App for restart loops and high memory usage. Then generate an az monitor scheduled-query create command to turn it into an alert rule.
   ```
